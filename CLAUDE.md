# Halabuda — складской учёт

Веб-приложение для складского учёта двух пространств (кунг). Работает в браузере Chrome на планшете Xiaomi Pad 6 (Android 13). Никакого сервера — всё локально через IndexedDB.

## Стек

- Vanilla HTML + CSS + JavaScript (без фреймворков)
- **Dexie.js 3.2.7** (`lib/dexie.min.js`) — обёртка над IndexedDB
- Открывается как файл `index.html` в Chrome

## Структура файлов

```
index.html          — каркас, все экраны и модальные окна
css/
  main.css          — глобальные стили, CSS-переменные, навигация, кнопки
  cabinet.css       — сетка ящиков, цветовая индикация
  modal.css         — bottom sheet, диалоги, поиск, история, отчёт
js/
  config.js         — схема шкафов (ряды, ящики, flex-пропорции) — НЕ в БД
  db.js             — Dexie БД, seed-данные, вспомогательные функции
  app.js            — инициализация, навигация между экранами
  cabinet.js        — отрисовка сетки шкафов, цвет ящиков
  drawer.js         — bottom sheet ящика: список позиций, +/−, редактирование
  history.js        — журнал операций, откат последних 10 действий
  search.js         — поиск по каталогу, создание/редактирование позиций
  report.js         — отчёт по остаткам (красные → жёлтые)
  more.js           — инвентаризация (с выбором режима), настройки ящиков
lib/
  dexie.min.js      — локальная копия Dexie.js
```

## База данных (Dexie / IndexedDB)

```
drawers  : { id, name }
items    : { ++id, name, min_stock, warn_coef }
stock    : { ++id, item_id, drawer_id, quantity }
history  : { ++id, timestamp, drawer_id, item_id, action, qty_before, qty_after }
```

Seed-данные (155 позиций, 25 ящиков) заливаются при первом запуске автоматически в `db.js → seedDatabase()`.

## Схема шкафов (config.js)

Два шкафа, layout задан статично — изменение только вручную в `config.js`.

**Основной шкаф** — 6 рядов:
- Ряд А: 2 ящика (1/2 + 1/2)
- Ряды Б, В: по 5 ящиков (равные)
- Ряды Г, Е: по 4 ящика (равные)
- Ряд Д: 2 ящика (2/3 + 1/3) — ящик 17 шире

**Кухня** — 4 ряда (один из них — пустая ячейка, `empty: true`)

## Цветовая логика ящика

```
min_stock == 0          → зелёный (контроль выключен)
qty < min_stock         → красный
qty < min_stock * coef  → жёлтый
иначе                   → зелёный
```

Если хоть одна позиция красная — ящик красный. Жёлтый приоритет ниже.

## Ключевые UX-решения

- **Поиск** — управление каталогом: создать новую позицию, редактировать имя/мин/коэф, видеть где что лежит
- **Ящик → Добавить позицию** — только выбор из существующего каталога (через поиск)
- **⋮ рядом с позицией** — изменить кол-во, мин. остаток, коэффициент; удалить из ящика
- **История** — последние 50 записей; кнопка откатa у первых 10
- **Инвентаризация** — три режима: все / жёлтые+красные / только красные

## Частые ошибки

- При сортировке по числовому ключу не использовать `|| default` — если значение `0`, оно считается falsy. Использовать `?? default` (nullish coalescing).

## Что не коммитить

- `lib/dexie.min.js` можно коммитить (локальная копия, нет npm)
- Никаких `node_modules`, `build/`, `.env` в проекте нет
